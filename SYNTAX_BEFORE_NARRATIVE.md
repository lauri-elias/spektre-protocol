# SYNTAX_BEFORE_NARRATIVE.md

**Status:** Core Ordering Constraint  
**Layer:** Decision Architecture / Human–AI Boundary  
**Invariant:** 1 = 1

---

## Definition

**Syntax Before Narrative** is a hard rule of system coherence:

> A system must establish **valid structure (syntax)** before allowing **meaning (narrative)** to form.

Narrative without syntax is fiction.  
Meaning without structure is manipulation.  
Action without syntax is undefined behavior.

---

## Thesis

Narrative explains.  
Syntax constrains.

When narrative precedes syntax, systems replace rules with stories and coherence with persuasion. This creates ambiguity, authority drift, and un-auditable decisions.

---

## The Ordering Rule

Syntax → State → Constraint → Commit → Narrative

Any inversion is invalid.

- **Syntax:** Structural correctness (rules, types, grammar).
- **State:** Explicit configuration of the system.
- **Constraint:** Invariants that cannot be violated.
- **Commit:** Irreversible execution.
- **Narrative:** Explanatory layer applied after execution.

---

## Why Narrative First Fails

Narratives are:
- Persuasive
- Contextual
- Power-amplifying

Syntax is:
- Deterministic
- Auditable
- Power-limiting

Allowing narrative first enables:
- Post-hoc justification
- Authority laundering
- Blame diffusion
- Rule bypass via storytelling

This is not a communication issue.  
It is a **control failure**.

---

## Common Violations

- Explaining outcomes before validating rules
- Storytelling to bypass constraints
- Policy by slogan instead of specification
- Moral framing without structural checks
- “Intent” claims without syntactic proof

These are **syntax errors**, not opinions.

---

## System-Level Consequences

When narrative precedes syntax:
- Rules become optional
- Enforcement becomes selective
- Responsibility becomes symbolic
- Trust becomes performative

At scale, this produces systems that persuade well and fail silently.

---

## Enforcement Protocol

### 1. Syntax Check
> “Is the structure valid?”

Types, rules, and interfaces must be explicit.

### 2. State Validation
> “What is configured right now?”

No narrative allowed before this step.

### 3. Constraint Enforcement
> “What cannot be broken?”

Invariants apply regardless of story.

### 4. Commit Gate
> “Is this irreversible?”

Ownership must be explicit.

### 5. Narrative (Optional)
Explanation may follow execution, never precede it.
